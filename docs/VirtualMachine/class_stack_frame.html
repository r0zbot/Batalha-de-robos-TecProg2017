Detailed Description

The StackFrame class is the collection of all data on the stack associated with programs subcalls.

The StackFrame class allow access to both function parameters, and automatic/local function variables.

When a function is called, a new partition is created at the current top pointer location. The StackFrame acts like a partition of the Machine data stack. All items from previous functions are in the "bottom" of the StackFrame, and should not be modified. Each current function has access to the remainder of the stack, from the StackFrame until the end of the stack page. The current function always has access to the "top" of the stack, and so functions do not need to take account of the memory usage of other functions or programs.

When the program enters a block such a function or method, space must be allocated on top of the StackFrame for the local variables inside the function scope. When the program leaves the block, the space must be freed again.
